## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

```bash

```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

# CRM Dashboard API Documentation

## Overview

This document provides an overview of the API endpoints available in the CRM Dashboard backend. The API is documented using Swagger and can be accessed at `/api/docs`.

## Users API

### Get All Users

- **Endpoint**: `GET /users`
- **Description**: Retrieve a list of all users.
- **Response**: Returns an array of user objects.

### Get User by ID

- **Endpoint**: `GET /users/:id`
- **Description**: Retrieve a user by their ID.
- **Response**: Returns a user object.
- **Error**: Returns 404 if the user is not found.

### Create a New User

- **Endpoint**: `POST /users`
- **Description**: Create a new user.
- **Request Body**: JSON object containing user details.
- **Response**: Returns the created user object.

### Update a User

- **Endpoint**: `PUT /users/:id`
- **Description**: Update an existing user.
- **Request Body**: JSON object containing updated user details.
- **Response**: Returns the updated user object.

### Delete a User

- **Endpoint**: `DELETE /users/:id`
- **Description**: Delete a user by their ID.
- **Response**: Returns the deleted user object.

## Authentication

All endpoints require authentication using JWT tokens. Include the token in the `Authorization` header as a Bearer token.

## Swagger Documentation

The API is documented using Swagger. You can view the API documentation by navigating to `/api/docs` in your browser.

## Environment Variables

Ensure the following environment variables are set:

- `MONGO_URI`: The connection string for MongoDB.

## Running the Application

To run the application, use the following command:

```bash
npm run start
```

Ensure that MongoDB is running and accessible at the URI specified in the environment variables.
